Vérification manuelle (Dashboard Supabase > Edge Functions > Secrets) :

create-checkout-session :
  STRIPE_SECRET_KEY
  SUPABASE_URL
  SUPABASE_SERVICE_ROLE_KEY
  STRIPE_PRICE_STRIPE_MONTHLY_AUTO_990    (Stripe Dashboard > Price ID)
  STRIPE_PRICE_PROMPTPAY_MONTHLY_990
  STRIPE_PRICE_PROMPTPAY_YEARLY_9900

stripe-webhook :
  STRIPE_SECRET_KEY
  STRIPE_WEBHOOK_SECRET
  SUPABASE_URL
  SUPABASE_SERVICE_ROLE_KEY

Stripe Dashboard > Webhooks > Endpoint, événements à écouter :
  checkout.session.completed
  invoice.paid
  invoice.payment_failed
  customer.subscription.updated
  customer.subscription.deleted
  payment_intent.succeeded
  payment_intent.payment_failed

Les 3 offres dans l’app correspondent aux codes :
  stripe_monthly_auto_990      (abonnement récurrent mensuel via Stripe)
  promptpay_monthly_990        (paiement one-shot mensuel via PromptPay)
  promptpay_yearly_9900        (paiement one-shot annuel via PromptPay, -2 mois)

Flux visiteur -> client payant :
  Landing -> "S'abonner" ou prix -> /inscription?plan=<code>&next=/app/abonnement
  -> signup email/mot de passe -> confirmation email -> /connexion
  -> connexion -> /app/abonnement?plan=<code>
  -> auto-checkout Stripe -> paiement -> /app/dashboard (espace privé RLS)

Supabase Auth (démo publique uniquement) :
  Authentication > Providers > Anonymous : Enabled
  (requis si VITE_PUBLIC_DEMO_MODE=true)

En production (VITE_PUBLIC_DEMO_MODE=false) :
  - /app/:section est protégée par <RequireAuth> (redirection /connexion).
  - testModeEnabled = false par défaut (l'app est verrouillée tant que pas d'abonnement actif).
